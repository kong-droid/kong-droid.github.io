성과

  • 실시간 데이터 중계 시스템 구축 - 데이터 전송 오류 0.1% 미만 달성
  • 첫 대형 고객사 유치 및 데이터 로거 220대 판매 기여

느낀 점

처음에는 로그 레벨을 낮춘 상태로 전송 테스트를 진행했기 때문에 중계 서버로의 데이터 전송 누락 문제를 인지하지 못했습니다.
베타 테스트 기간이 얼마 지나지 않아 고객사 개발팀과 협업하면서 일부 온도 데이터가 100% 전송되지 않고 누락되는 문제를 발견하게 되었습니다.
에러 현상을 재현하기 위해 화이트 박스 테스트와 블랙 박스 테스트를 병행해 진행한 결과, 스마트팟 서버에서 중계 서버로 동기 통신 방식으로 데이터를 전송한 뒤 고객사 서버로 전송할 때 타임아웃 오류가 발생한다는 점을 확인할 수 있었습니다.
분석 결과 '스마트팟 서버가 중계 서버로 데이터를 반드시 전송해야 하지만, 응답값은 필요하지 않다'는 특성을 고려해 webClient 클래스를 활용해 중계 서버에 온도 데이터를 전송했습니다.
에러 수정 후 운영 서버를 구축하고 배포한 결과, 고객사 서버에 에러 없이 온도 데이터가 정상적으로 전송되는 것을 확인할 수 있었으며, 언제나 오류는 발견되지만 원인 분석을 위해 여러 방면에서 생각한다면 수정할 수 있다를 깨달았습니다.
