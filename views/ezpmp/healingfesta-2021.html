프로젝트 기술서

주요업무

  • 비동기 통신을 활용해 오투미트 API 통신 후 로그인, 회원가입, 게시판, 설문 기능 데이터 호출 및 화면 제작
  • 모바일 결제 모듈 제작
  • 테스트 및 디버깅

상세 업무

  • 회원가입
    • 정규표현식을 활용해 아이디, 이메일, 비밀번호와 같은 필수값 검증 기능 제작
    • 오투미트 문자 인증 API와 통신해 이메일 본인 인증 기능 제작
  • 로그인
    • localStorage 내 로그인 토큰을 저장하고 서버 측과 통신해 로그인 유지 기능 제작
  • 게시판
    • 관리자 웹 게시글 작성/수정/삭제 : 오투미트 게시글 관리 API와 통신 및 recoil.js를 활용해 게시 목록, 게시글 인덱스 별 상태 값을 관리함
    • 사용자 웹 : 게시판별 게시글 목록 조회 기능 제작
    • 오투미트 파일 첨부 API와 통신해 게시글 별 파일 첨부 기능 제작
  • 설문
    • 관리자 웹 설문 작성/수정/삭제 : 오투미트 설문 관리 API와 통신 및 recoil.js를 활용해 게시 목록, 게시글 인덱스 별 상태 값을 관리함
    • 관리자 웹 설문 결과 작성/수정/삭제 : 오투미트 설문 관리 API와 통신을 통해 설문 점수 범위를 지정

성과

  • 관리자 웹의 게시판, 설문, 회원 정보 조회 기능 완성
  • 사용자가 모바일에서 강의 결제 및 취소를 오류 없이 원활하게 진행함

느낀 점

처음으로 React와 결제 모듈을 구현해 본 프로젝트였습니다.
오투미트 API를 활용해 웹을 제작하면서 비동기 통신의 개념과 ES6 문법을 익힐 수 있었고, 결제 기능을 처음 구현하던 당시, 비동기 통신을 사용하면서 '동일 출처 정책(Same-Origin Policy)' 문제에 부딪히기도 했습니다.
그 과정에서 ‘여러 프로젝트에서 재사용하기 쉬운 방식으로 만들 수는 없을까?’라는 아이디어가 떠올랐고, 하나의 결제 모듈로 결제의 시작과 끝을 마무리하고 웹에서는 결제 결과만 표시하도록 구성했을 때의 짜릿함은 아직도 생생하게 기억납니다.
시나리오를 직접 구성하고 개발을 마친 뒤, 여러 번의 테스트를 거쳐 실제 운영 환경에서 결제가 오류 없이 원활하게 이루어졌을 때, 처음으로 저 자신에게 개발자로서의 가능성을 본 것 같습니다.
