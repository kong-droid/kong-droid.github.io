프로젝트 기술서

주요업무

  • 테스트 서버 및 운영 서버 무중단 배포
  • 행사 정보, 설문, 상품, 파일 첨부, 카테고리 DBMS 설계
  • 행사 정보, 설문, 상품, 파일 첨부 API 개발
  • 행사 정보 관리자 웹 개발

상세 업무

  • 로그인
    • 쿠키에 로그인 토큰을 저장하고 서버 측과 통신해 로그인 유지 기능 제작
    • 토큰을 서버에 리다이렉트 후 유효성을 검증하는 SSO 기능 제작
  • 설문
    • 설문 유형 및 행사 코드, 설문 인덱스 기준으로 사용자 웹의 행사 별 공통 설문, 고객 간의 비즈니스 미팅 설문 기능 제작
    • 관리자 웹에서 설문 API와 비동기 통신을 통해 회원, 비회원의 설문 결과 집계 목록 및 엑셀 다운로드 제작
    • 저장/수정/삭제 : 설문 기능을 관리자 및 사용자 웹에 비동기 통신을 통해 데이터 제어 및 관리
  • 상품
    • 상품 유형 및 행사 코드에 따라 상품을 그룹화
    • 다국어 설정을 위해 상품 공통 정보 테이블과 언어별 설정 테이블 설계 (1:N)
  • 행사 정보
    • 관리자 웹 화면 설계 및 행사 기본 정보, 헤더 메뉴, 내비게이션 바, 푸터, 행사 사용 언어, 행사 상세 옵션 기능 제작
    • 메인 진입 시 데이터베이스의 키워드 값을 통해 도메인 검증을 통해 행사장으로 이동 기능 제작
    • 다국어 설정을 위해 행사 정보 공통 테이블과 언어별 설정 테이블 설계 (1:N)
    • 저장/수정/삭제 : 행사 정보 API를 관리자 웹에 비동기 통신을 통해 데이터 제어 및 관리
  • 통계
    • 서브 쿼리에서 조인으로 변경해 쿼리 성능 개선

성과

  • 관리자 웹 접속 통계, 영상 통계 조회 쿼리를 40초 1초로 개선 및 엑셀 다운로드 구현해 집계 정보를 제출을 쉽게 함
  • 설문 API 개발 및 개발 문서 공유를 통해 행사별 원하는 설문을 생성 후 회원, 비회원의 선택 값으로 가중치를 집계해 행사 별로 2023년 행사를 대비할 수 있게 분석하고 문서 작성을 용이하게 함
  • 파일 첨부 시 물리 저장소 경로를 /행사명/년월일/카테고리 별로 규칙을 부여하고 일관성 있는 데이터를 제공
  • 관리자 웹을 통해 사용자가 행사 취지에 맞게 헤더, 푸터, 네비게이터바, 메뉴, 세부 옵션값을 관리할 수 있게 함
  • 하나의 서버에 도메인에 따라 각각 다른 행사로 이동해 리소스 효율성을 높임

느낀 점

프로젝트를 진행하며 CSR, SSR, MPA, SPA, Cookie, Session 등 프론트엔드의 기본 개념과 함께 ES5, ES6 문법을 익힐 수 있었습니다.
또한 JSP를 다뤄보며 JSP의 동작 방식은 물론, 동기 통신과 비동기 통신의 차이도 체감하며 배울 수 있었습니다.
이 외에도 온프레미스 서버를 구축하고 유지보수하는 과정을 통해 서버 관련 지식도 자연스럽게 습득하는 과정에서 하나의 서버에서 여러 도메인을 포트 포워딩으로 연결해 사용자가 원하는 메인 페이지로 이동시키는 방법을 직접 고민하고 구현해보며 세상에는 다양한 효율적인 방식이 존재하고 나도 할 수 있다는 자신감을 얻게 된 소중한 경험이었습니다.
